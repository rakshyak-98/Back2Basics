gRPC is an RPC framework operating fundamentally over HTTP/2, utilizing Protocol Buffers (Protobuf) as both its Interface Definition Language (IDL) and underlying message interchange format.

> gRPC discards JSON/XML in favor of binary Protobuf uses a tag-based encoding. It does not transmit field names.

- Deserialization is extremely fast compared to JSON due to direct memory mapping capabilities and lack of string parsing.
- CPU cost is heavily shifted toward bitwise operations (varint decoding) rather than lexical analysis.

## Interface Definition (Protobuf IDL)

As a backend developer, your source of truth is the `.proto` file. You do not write routing boilerplate; you define the contract and generate the server stubs.

### API contract implementation Details 

- Versioning: Package names must include versions to prevent breaking changes (e.g., `package api.v1;`)
- Well-Know Type (WKT): Utilize `google/protobuf` packages for common types instead of primitives where nullable or semantic data is required.

```protobuf
syntax = "proto3";
package inventory.v1;

option go_package = "github.com/org/repo/internal/gen/inventory/v1";

import "google/protobuf/timestamp.proto";
import "google/type/money.proto";

service InventoryAPI {
  rpc ReserveStock(ReserveStockRequest) returns (ReserveStockResponse);
}

message ReserveStockRequest {
  string idempotency_key = 1;
  string sku = 2;
  int32 quantity = 3;
}

message ReserveStockResponse {
  string reservation_id = 1;
  google.protobuf.Timestamp expires_at = 2;
}
```

### Server Implementation & Context Management

you must implement the interface generated by `protoc`. The most critical operational parameter here is the `Context` object passed to every RPC method.

**Context LifeCycle Mechanics:**
- Deadline Propagation: The client dictates the timeout via `grpc-timeout`. This is automatically parsed into `Context` deadline.
- Short-circuiting: You must monitor the `Context` for cancellation `ctx.Done()`. If the client drops the connection or the deadline is exceeded, halt expensive database queries or downstream API calls immediately.

**Concrete Primitive: Go server Implementation**

```go
func (s *InventoryServer) ReserveStock(ctx context.Context, req *pb.ReserveStockRequest) (*pb.ReserveStockResponse, error) {
    // 1. Check for client cancellation or timeout before starting work
    if err := ctx.Err(); err != nil {
        return nil, status.Errorf(codes.Canceled, "client cancelled: %v", err)
    }

    // 2. Pass context to downstream datastore to inherit the gRPC deadline
    res, err := s.db.ExecContext(ctx, "UPDATE stock SET qty = qty - $1 WHERE sku = $2", req.Quantity, req.Sku)
    if err != nil {
        // Handle error (see Section 3)
    }

    return &pb.ReserveStockResponse{
        ReservationId: generateUUID(),
        ExpiresAt:     timestamppb.Now(),
    }, nil
}
```

> [!NOTE]
> Do not return standard HTTP status codes or generic language errors. gRPC dictates a strict error model using `google.golang.org/grpc/codes` and `status.Status`
> - Map internal domain errors to the 16 standard gRPC codes `codes.NotFound` `codes.Unauthenticated` `codes.InvalidArgument`
> - Error Details -> Attach machine-readable Protobuf messages to the error trailer for structured client-side handling using `status.WithDetails()`


**Concrete Primitive: Returning Rich Errors**

```go
import (
    "google.golang.org/grpc/codes"
    "google.golang.org/grpc/status"
    "google.golang.org/genproto/googleapis/rpc/errdetails"
)

// Inside RPC handler:
st := status.New(codes.InvalidArgument, "invalid stock quantity")
v := &errdetails.BadRequest_FieldViolation{
    Field:       "quantity",
    Description: "Quantity must be greater than zero",
}
st, _ = st.WithDetails(&errdetails.BadRequest{FieldViolations: []*errdetails.BadRequest_FieldViolation{v}})

return nil, st.Err()
```

### Interceptors (Middleware)

Cross-cutting (authentication, logging, tracing, metrics) are implemented via interceptors.
- Unary interceptors -> Wrap a single request/response.
- Stream Interceptors: Wrap the stream establishment and message payload events.

```go
func AuthInterceptor(ctx context.Context, req interface{}, info *grpc.UnaryServerInfo, handler grpc.UnaryHandler) (interface{}, error) {
    // 1. Extract metadata (HTTP/2 headers)
    md, ok := metadata.FromIncomingContext(ctx)
    if !ok {
        return nil, status.Errorf(codes.Unauthenticated, "missing metadata")
    }

    // 2. Validate token
    tokens := md.Get("authorization")
    if len(tokens) == 0 || !validateJWT(tokens[0]) {
        return nil, status.Errorf(codes.Unauthenticated, "invalid token")
    }

    // 3. Inject parsed identity into context and invoke actual handler
    newCtx := context.WithValue(ctx, "userID", "123")
    return handler(newCtx, req)
}
```
